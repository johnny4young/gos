# Code of Conduct

`gos` is maintained for developers who want a simple, reliable way to install
and switch Go versions. The project should be useful to newcomers and
experienced maintainers alike.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but ask for evidence when debugging.
- Keep feedback focused on the code, docs, reproducible behavior, and user
  impact.
- Help newcomers produce actionable bug reports and small pull requests.
- Respect maintainer time by keeping issues scoped and reproducible.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private information, credentials, machine-specific secrets, or
  sensitive vulnerability details.
- Repeated off-topic comments, spam, or demands for unpaid support.
- Intentionally misleading reports, unsafe install instructions, or exploit
  details in public threads.

## Reporting

For community conduct concerns, open a private maintainer contact request
through GitHub without posting sensitive details publicly.

For security vulnerabilities, follow [SECURITY.md](SECURITY.md) and use GitHub
Security Advisories.

Maintainers may edit, hide, lock, or remove comments and may block participants
who violate these standards.
