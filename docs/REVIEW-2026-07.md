# Deep Project Review — July 2026

A full audit of gos covering security, correctness, performance, architecture,
maintainability, release engineering, and test coverage. Every file in the
repository was reviewed: `gos.sh`, both installers, completions, packaging
metadata, release scripts, workflows, and the test suite.

This document records (1) what was found and fixed in the accompanying PR,
(2) what was found and deliberately deferred, and (3) a roadmap of the changes
that would take gos from a solid tool to a world-class one.

---

## 1. Fixed in this PR

### Correctness bugs (all reproduced before fixing)

| Bug | Impact |
|---|---|
| `gos use` / `gos install 1.22` failed for bare `X.Y` versions | Since Go 1.21, the first release of a minor ships as `X.Y.0`, so the extremely common `go 1.22` go.mod directive produced a 404. Bare minors now resolve to the newest matching patch release from the feed. |
| A broken `go` on PATH aborted gos entirely | Under `set -e`, a `go` binary that fails (wrong arch, corrupt install) killed `gos current`, `gos install`, and — worst — `gos latest`, the very command that could repair it. `_gos_current` is now failure-tolerant. |
| Trailing slash in `GOS_INSTALL_DIR` broke every upgrade | `/usr/local/go/` computed backup/rollback paths *inside* the install tree, so `mv` failed. Now normalized at startup. |
| `gos list` misordered pre-releases | `sort -t. -kN,Nn` treated `1.24rc2` as `1.24`, interleaving rc/beta among patch releases and above final releases. Replaced with a semantic decorate-sort-undecorate (`beta < rc < release`). |
| `gos latest` printed `Current: gonone` on machines without Go | Cosmetic but embarrassing first-run output. |
| `gos doctor` false negatives | Hashing `$0` broke under `curl \| bash -s doctor`; the Windows PATH-order pattern (`*\\go.exe`) could never match Git Bash's forward-slash paths; the completions check did not resolve symlinked installs. |
| `gos platforms` fallback parser emitted bogus entries | Without `jq`/`python3`, `go1.x.src.tar.gz` leaked through as a "platform" (including into `--json` output). Now filtered to `os/arch` pairs. |
| Last-resort feed parser dropped rc/beta versions entirely | The character class excluded letters; widened. |
| `--json` inconsistency | `gos prune --json` was a hard error while `gos --json prune` was silently ignored. `prune` now supports `--json` output. |
| `gos install 1.21.6 --json` silently discarded trailing args | Now rejected with a usage error. |
| Manifest at filesystem root ignored | `_gos_find_upward` stopped before checking `/`. |

### Security hardening

| Finding | Fix |
|---|---|
| `GOS_INSTALL_DIR` denylist bypass via `..` | `/usr/local/../../etc/gogo` passed the textual system-path denylist and could steer privileged `rm -rf`/`mv` outside the intended tree. `.`/`..` components and control characters are now rejected. |
| Pre-release tags became `releases/latest` | A `v1.6.0-rc1` release would have been served to every stable `install.sh` user and pushed to the Homebrew tap. Releases with `-` in the version are now marked pre-release and skip the tap publish. |
| Tag-push releases did not verify the tag was on `main` | Any branch commit tagged `v*` ran the full release with `contents: write`. The workflow now requires the tagged commit to be an ancestor of `main`. |
| Silent `sed` no-ops could ship unpinned installers | If the patch anchors in `install.sh`/`install.ps1` drifted, the release published installers that silently skipped checksum verification. Patches are now grep-verified after application. |
| Same-origin `.sha256` fallback under-documented | The companion-file fallback shares the archive's trust boundary; SECURITY.md now states this explicitly and clarifies that the cross-origin feed path is the stronger check. |

New security-relevant features were designed fail-closed: `GOS_DOWNLOAD_MIRROR`
refuses to download when no official checksum is available or no SHA256 tool
exists, and `gos self-update` verifies the release `checksums.txt`, runs
`bash -n`, and refuses to overwrite Homebrew or git-checkout installs.

### Release engineering

- `git push --atomic origin main "$TAG"` — a half-pushed release (main updated,
  tag missing) previously required manual repair.
- `concurrency: group: release` — overlapping release runs no longer race.
- The version-bump job asserts `GOS_VERSION` matches the tag, and the release
  smoke test asserts `./gos.sh version` equals the tag (now also on Windows).
- The Windows zip pins `gos.sh`'s exec bit so its checksum cannot drift with
  checkout file modes.
- Homebrew tap pushes retry with a rebase (shared-tap races between sibling
  repos).
- `update-changelog.bash` is now bash 3.2-safe (macOS empty-array expansion).
- The winget manifest gained the schema-required `PackageLocale` and
  `NestedInstallerType`/`NestedInstallerFiles` fields, so it can actually be
  submitted.
- Removed `packaging/chocolatey/tools/gos.cmd`, a dead drift-prone duplicate
  (the shim actually installed comes from the release zip).

### New features

- **`gos check [--json]`** — update availability without installing; designed
  for CI and prompts/statuslines.
- **`gos self-update`** — checksum-verified, syntax-checked in-place upgrade of
  gos itself, with guards for package-manager-owned installs.
- **`GOS_DOWNLOAD_MIRROR`** — HTTPS-only archive mirror support for
  restricted-network users, with mandatory verification against go.dev
  metadata (a mirror can deny service, never tamper undetected).
- **Smarter completions** — context-aware flags (`--json` only where valid,
  `--rollback` after `prune`), directory completion for `gos use`, proper zsh
  scoping (no more leaking `$state` globals into interactive shells).

### Test coverage added

`tests/features.bash` grew from 16 to 29 scenarios: check text/JSON, mirror
(verified download, unverifiable fail-closed, plaintext rejection),
self-update (replace, idempotent, git-checkout guard), bare-minor resolution,
broken `go` tolerance, corrupted-cache re-download, prune `--json`, rollback
error paths, roll-forward, trailing-slash and `..` install-dir guards, and rc
ordering in `list`. `tests/install-sh.bash` now also exercises the
release-pinned checksum path (verify + mismatch abort) that every real
release-asset user runs.

## 2. Follow-ups (status after the second pass)

All items in this section except the first were **implemented in this same
PR** in a follow-up pass:

- **Pin GitHub Actions to commit SHAs** — *still open, intentionally.*
  `softprops/action-gh-release@v3` runs with `contents: write` + attestation
  permissions; a repointed tag is a real supply-chain vector. The commit SHAs
  could not be verified from this environment, and pinning to an unverified
  SHA would be worse than a tag. To do it: for each `uses:` entry run
  `gh api repos/<owner>/<repo>/git/ref/tags/<tag>` (dereference annotated
  tags with `^{}`), replace the tag with the 40-hex SHA plus a
  `# vN` comment, and let Dependabot (already configured for
  `github-actions`) keep the pins fresh.
- ~~Pin GitHub's SSH host keys~~ — **done**: `update-homebrew-tap.sh` now
  builds a `known_hosts` from the GitHub meta API over TLS (an independent
  trust channel) and uses `StrictHostKeyChecking=yes`, falling back to
  `accept-new` with a warning only if the API is unreachable.
- ~~Same-origin fallback under strict mode~~ — **done** as an opt-in third
  level: `GOS_REQUIRE_CHECKSUM=feed` requires the cross-origin feed digest
  and rejects the `.sha256` fallback; `=1` keeps its documented meaning so
  no-`jq` environments don't break.
- ~~Interrupted-activation window~~ — **done**: the EXIT trap restores the
  backed-up installation if gos dies between the backup `mv` and the
  activation `mv`, and `gos prune` now reports orphaned
  `*.gos-backup.<pid>`/`*.gos-current.<pid>` residue (removing it with
  `--rollback` when the active install is healthy).
- ~~`_gos_sudo` stream merging~~ — **done**: stdout and stderr are captured
  separately; warnings go to stderr.
- ~~`Add-UserPath` flattens `REG_EXPAND_SZ`~~ — **done**: both `install.ps1`
  and `uninstall.ps1` read/write the user `Path` through
  `Microsoft.Win32.Registry` preserving the value kind (and unexpanded
  `%VAR%` entries), then broadcast `WM_SETTINGCHANGE`.
- ~~Idempotent short-circuit trusts PATH `go`~~ — **done**: the no-op path
  now also requires `GOS_INSTALL_DIR` to actually serve the version (or PATH
  `go` to resolve from inside it).
- ~~Heading-only Unreleased silently discarded~~ — **done**: the changelog
  helper errors out instead of falling back to git subjects.
- ~~Functional tests for `update-homebrew-tap.sh`~~ — **done**:
  `tests/homebrew-tap.bash` runs the publish flow against a local bare tap
  (via the `GOS_TAP_REMOTE` hook), covering regeneration, idempotency,
  placeholder/malformed-sha rejection, and fork behavior. Writing it
  surfaced and fixed a real first-publish bug (missing `Formula/` directory
  in a fresh tap).

## 3. Roadmap to world-class

### Architecture

gos's core identity — *one auditable Bash script, no runtime, no shims* — is
its moat. The recommendations below deliberately preserve it.

1. **Side-by-side versions with instant switching — SHIPPED (opt-in).**
   Set `GOS_VERSIONS_DIR` and each version installs to
   `$GOS_VERSIONS_DIR/go<version>` with `GOS_INSTALL_DIR` as a symlink to the
   active one: `gos use`/`gos install` of an installed version is an instant
   re-link with zero network, and `gos list --installed` / `gos uninstall`
   manage the set. The classic in-place layout remains the default.
   Remaining v2.0 work: make it the default in a major release, and a
   Windows-native strategy (Git Bash symlinks are copies, so the mode is
   documented as macOS/Linux/WSL).

2. **Split source, ship single file.** ~1500 lines in one file is nearing the
   maintainability ceiling. Develop as `src/*.sh` modules concatenated by a
   build step (with a CI check that the shipped `gos.sh` matches the build) —
   contributors get structure, users still get one file. Adopt only when the
   file grows past ~2000 lines.

3. **Structured internals.** Progress/log lines currently share stdout with
   data output. Move human chatter to stderr (keeping stdout pure for data or
   JSON) — it makes every command scriptable, not just the `--json` ones.

### Product / features

4. **`gos env` — SHIPPED** (`eval "$(gos env)"`, `gos env --fish | source`,
   `--json`). Future: per-shell overrides (`GOS_GO_VERSION=1.22 gos env`)
   once side-by-side is the default.
5. **Auto-switch on `cd` (opt-in)** — a tiny shell hook that runs `gos use
   --quiet` when entering a directory with `.go-version`/`go.mod`. This is
   the feature users of asdf/mise expect most.
6. **`gos exec <version> -- <cmd>`** — run one command under a specific Go
   without switching, ideal for CI matrices and bisecting regressions.
7. **Signature verification beyond checksums** — verify GitHub artifact
   attestations for gos's own releases (`gh attestation verify`) when `gh` is
   available, and document the manual path.
8. **Native Windows (PowerShell) port of the core** — today Windows requires
   Git Bash. A `gos.ps1` implementing latest/install/use would unlock the
   winget/Chocolatey channels for the majority of Windows devs.
9. **Richer `doctor`** — `GOPATH`/`GOROOT` sanity, stale rollback size,
   proxy detection (`GOPROXY`, corporate MITM certs), disk space preflight.

### Distribution

10. **Submit the winget and Chocolatey packages** (metadata is now
    schema-valid); add a Scoop manifest — Scoop is trivial to maintain and
    popular with Go developers on Windows.
11. **Publish to the Homebrew tap with bottles disabled but completions
    installed** — the formula should `bash_completion.install` the three
    completion files so brew users get tab-completion out of the box.
12. **Container/CI story** — a documented one-liner layer for Dockerfiles and
    a tiny GitHub Action (`uses: johnny4young/setup-gos@v1`) that runs
    `gos use` against the repo's manifest. CI is where pinned Go versions
    matter most; meeting users there drives adoption.

### Engineering hygiene

13. **Extract shared test fakes** — the ~50-line fake `curl`/`sha256sum`
    scaffolding is copy-pasted across five test files; move to
    `tests/lib/fakes.bash`.
14. **Single source of truth for duplicated constants** — the semver regex,
    repo slug, and `UPDATE_ON_RELEASE` sentinel each live in 3–8 files; at
    minimum add invariant tests asserting the copies match.
15. **Nightly canary workflow — SHIPPED** (`.github/workflows/canary.yml`):
    exercises `check`, `list`, `platforms`, pinned install with
    `GOS_REQUIRE_CHECKSUM=feed`, upgrade, rollback, and side-by-side mode
    against the live go.dev feed on all three OSes, nightly and on demand.
16. **Coverage discipline — SHIPPED for the known gaps**: rollback
    validation-failure restore, `_gos_store_cache` write-failure warning,
    and `gos current --json` with no working Go are now covered
    (37 scenarios in `tests/features.bash`).

### Suggested sequencing

| Phase | Items | Theme |
|---|---|---|
| Next release | SHA-pin actions (needs network to verify), fakes extraction (13), constants invariants (14) | Close remaining audit items |
| v2.0 | Side-by-side by default + `GOS_GO_VERSION` per-shell overrides | Become a true version manager by default |
| v2.x | cd auto-switch (5), `gos exec` (6), setup-gos action (12) | Daily-driver ergonomics |
| Parallel track | winget/choco/scoop submissions (10), PowerShell port (8) | Windows reach |
